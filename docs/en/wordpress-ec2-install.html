Installing WordPress on EC2

Published on 2020/02/17 📖 2 min read
WordPress AWS EC2

Introduction

A friend teased me, saying “You work in IT, so you can at least build a website, right?” So I decided to install WordPress on an AWS EC2 instance.

The WordPress installation requirements are as follows:

Requirements | WordPress.org https://wordpress.org/about/requirements/

  • PHP version 7.3 or greater.
  • MySQL version 5.6 or greater, OR MariaDB version 10.1 or greater
  • HTTPS support

Environment Information

I’ll be installing on a free-tier t2.micro instance running Amazon Linux.

MySQL

Installing MySQL

Amazon Linux comes with MariaDB installed by default, so we’ll configure it to use MySQL 5.7 instead. If MariaDB is acceptable, no additional configuration is needed.

sudo su -
yum info mysql

#リポジトリの追加
yum localinstall -y https://dev.mysql.com/get/mysql80-community-release-el7-1.noarch.rpm

#mysql8.0リポジトリの無効化
yum-config-manager --disable mysql80-community

#mysql5.7リポジトリの有効化
yum-config-manager --enable mysql57-community

#mysql5.7がインストールできるか確認
yum info mysql-community-server

#インストール
yum -y install mysql-community-server

#バージョン確認
mysqld --version

#自動起動確認
systemctl start mysqld.service
systemctl enable mysqld.service
systemctl status mysqld.service

With Apache installed and enabled, let’s do a test access to the web server. Enter the public IP address in your browser and navigate to it. Since no content has been deployed yet, the Apache test page should be displayed.

Note: You need to configure the AWS Security Group accordingly. Make sure to open the HTTP (or HTTPS) protocol and port.

Changing File Permissions

The Amazon Linux Apache document root is /var/www/html, which is owned by root by default. We need to change this so that the ec2-user can also edit files. Specifically, we’ll add the ec2-user to the apache group, assign ownership of the /var/www directory to the apache group, and grant write permissions to the group. This will allow editing of WordPress files and PHP files.

usermod -a -G apache ec2-user
exit

Log back in and verify.

[ec2-user@wp-ec2 ~]$ groups
ec2-user adm wheel apache systemd-journal
[ec2-user@wp-ec2 ~]$

Change the group ownership of /var/www and its contents to the apache group.

sudo chown -R ec2-user:apache /var/www

Add group write permissions and set the group ID on future subdirectories by changing the directory permissions of /var/www and its subdirectories.

sudo chmod 2775 /var/www && find /var/www -type d -exec sudo chmod 2775 {} \;

Recursively change the file permissions of /var/www and its subdirectories to add group write permissions.

sudo find /var/www -type f -exec sudo chmod 0664 {} \;

Modifying httpd.conf

vi /etc/httpd/conf/httpd.conf

Change AllowOverride None to AllowOverride All.

WordPress

Installing and Configuring WordPress

We’ll set aside the SSL/TLS configuration for now and proceed with installing and configuring the WordPress environment.

wget https://ja.wordpress.org/latest-ja.tar.gz
tar -xzf latest-ja.tar.gz

Creating and Editing the wp-config.php File

cp wordpress/wp-config-sample.php wordpress/wp-config.php
vi wordpress/wp-config.php

Enter the following information:

define('DB_NAME', 'wordpress-db');
define('DB_USER', 'wordpress-user');
define('DB_PASSWORD', 'your_strong_password');

Next, modify the Authentication Unique Keys and Salts section. These KEY and SALT values provide a cryptographic layer for the browser cookies stored on users’ local machines by WordPress. Essentially, specifying long and random values here improves your site’s security.

define( 'AUTH_KEY',         'put your unique phrase here' );
define( 'SECURE_AUTH_KEY',  'put your unique phrase here' );
define( 'LOGGED_IN_KEY',    'put your unique phrase here' );
define( 'NONCE_KEY',        'put your unique phrase here' );
define( 'AUTH_SALT',        'put your unique phrase here' );
define( 'SECURE_AUTH_SALT', 'put your unique phrase here' );
define( 'LOGGED_IN_SALT',   'put your unique phrase here' );
define( 'NONCE_SALT',       'put your unique phrase here' );

Access the following URL to generate SALT keys and paste them in:

https://api.wordpress.org/secret-key/1.1/salt/

Changing Permissions

Change the owner and group of everything under the wordpress directory to apache.

sudo chown -R apache:apache /var/www/html/*

WordPress Setup

Access the following URL and enter the WordPress installation settings. Replace xxx.xxx.xxx.xxx with the Elastic IP attached to the EC2 instance.

http://xxx.xxx.xxx.xxx/wp1/wp-admin/install.php

At this point, WordPress is up and running. Custom domain configuration and HTTPS setup will be covered separately.
