How to Configure Metabase Email Settings (Dashboard Subscriptions) Using Gmail

Published on 2021/09/12 üìñ 1 min read
Metabase

Metabase allows you to choose between Slack and email when sharing dashboards. I’ll summarize the settings and configuration method for using Gmail for email. I don’t think there’s much demand for sending via Gmail for business purposes. Setting up your own mail server at the personal level can be quite a hassle, so being able to use Gmail is convenient.

Reference Materials

Dashboard Subscriptions https://www.metabase.com/docs/latest/users-guide/dashboard-subscriptions.html

Setting Up Email https://www.metabase.com/docs/latest/administration-guide/02-setting-up-email.html

Setup Screen

Input Values

SMTP Host: smtp.gmail.com
SMTP Port: 587
SMTP Security: TLS
SMTP Username: Gmail account (email@gmail.com)
SMTP Password: Gmail password

Other Settings Required

When I configured the above values in Metabase, it failed. The reason is that it gets classified as a less secure app by Gmail. Check this article and enable access for less secure apps. As a precaution, I created a dedicated Gmail account for sending rather than using my regular Gmail account.

https://support.google.com/accounts/answer/6010255?hl=ja

Less secure apps & your Google Account

With this done, the email configuration is complete. After that, configure the settings according to the options and send the email.
