processor : 1

Method 1: Disabling Hyper-Threading

Run the following command. In this environment, there is cpu0 and cpu1, and we disable CPU1. In this case, hyper-threading is disabled but reverts after a reboot. Running the same command at startup is an option, but requires some effort such as creating a script.

sudo su -
echo 0 > /sys/devices/system/cpu/cpu1/online

Method 2: Disabling Hyper-Threading

Optimizing CPU options - Amazon Elastic Compute Cloud https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/instance-optimize-cpu.html#instance-cpu-options-rules

As stated in the manual above, CPU options can only be selected at instance creation time. For instance types that support this feature, a selection screen like this appears when creating the instance:

Setting threads per core to 1 here will disable hyper-threading. You can also specify this via the AWS CLI using --cpu-options as follows:

aws ec2 run-instances --image-id ami-xxxxxxxx --instance-type m5.large --cpu-options "CoreCount=1,ThreadsPerCore=1" --key-name keypair

Verify the settings after launch:

[ec2-user@bastin ~]$ aws ec2 describe-instances --instance-ids i-01842d7da54e0c630 --query "Reservations[].Instances[].CpuOptions"
[
    {
        "CoreCount": 1,
        "ThreadsPerCore": 1
    }
]

Other Notes

Amazon EC2 Instance Types

https://aws.amazon.com/ec2/instance-types/

Except for M6g instances, A1 instances, T2 instances, and m3.medium, each vCPU is a thread on an Intel Xeon core or an AMD EPYC core.

As stated above, some instances have hyper-threading disabled by default. Note that “hyper-threading” is an Intel term and doesn’t apply to AMD, which is why “threads” is used instead.
