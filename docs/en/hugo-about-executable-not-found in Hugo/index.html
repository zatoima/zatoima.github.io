About the Executable Not Found Error in Hugo

2021/08/31Published on 📖 1 min read
🦔Hugo

Error

An error occurs indicating that the Go executable file cannot be found in the PATH environment variable.

D:\hugo\zatoima.github.io>hugo server
Error: failed to download modules: exec: "go": executable file not found in %PATH%

Solution

The official documentation has information about this.

Error: Go executable not found

Install Hugo’s Go dependency . If you believe that Go is already installed, perhaps Hugo was unable to detect Go due to installing either Go or Hugo in an isolated way such as via Linux Snaps. Users on all major platforms have reported success installing Hugo and Go by following the official Wowchemy installation guide . You can also browse or report issues with Hugo detecting Go on the Hugo Forum .

On Windows, running the PowerShell commands described in this official documentation resolved the error.

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
iwr -useb get.scoop.sh | iex

scoop install git go hugo-extended
