Importing Morphological Analysis Data of Aozora Bunko Works into RDS PostgreSQL

Published on 2020/05/17 📖 2 min read
PostgreSQL AWS Aurora RDS

Introduction

I wanted to perform analysis using morphological analysis data on PostgreSQL, so here’s a note on how to load it into RDS (PostgreSQL) on AWS. I used this morphological analysis data:

Aozora Bunko Morphological Analysis Dataset http://aozora-word.hahasoha.net/

Target Data

I decided to load the file containing all data into PostgreSQL.

http://aozora-word.hahasoha.net/utf8/utf8_all.csv.gz

This file contains all Aozora Bunko texts as pre-parsed morphological data in the following format. It has 87,701,673 rows and a file size of approximately 6GB.

Create Table on PostgreSQL Side

Rough data types for simplicity:

CREATE TABLE aozora_kaiseki(file VARCHAR(30),num INT,row INT,word TEXT,subtype1 VARCHAR(30),subtype2 VARCHAR(30),subtype3 VARCHAR(30),subtype4 VARCHAR(10),conjtype VARCHAR(15),conjugation VARCHAR(15),basic TEXT,ruby TEXT,pronunce TEXT );
Load with COPY Command

Download the data with wget:

wget http://aozora-word.hahasoha.net/utf8/utf8_all.csv.gz
gzip -d utf8_all.csv.gz

Load with the copy command:

psql -h rdspgsqlv1.xxxxxx.ap-northeast-1.rds.amazonaws.com -d postgres -U postgres -c "COPY aozora_kaiseki(file,num,row,word,subtype1,subtype2,subtype3,subtype4,conjtype,conjugation,basic,ruby,pronunce) from stdin with csv DELIMITER ','" < /home/ec2-user/utf8_all.csv
