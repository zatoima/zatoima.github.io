Downtime Log When Applying a Patch to Aurora PostgreSQL

Published on 2022/07/20 üìñ 1 min read
AWS Aurora

This is for reference only, as it depends on the patch content. In this case, the actual downtime during patch application was approximately 7 seconds.

Script

import psycopg2
import psycopg2.extras
import time
import datetime

for i in range(20000):
  try:
    dt_now = datetime.datetime.now()
    conn = psycopg2.connect("host=auroraserverlessv2.cluster-cm678nkt5thr.ap-northeast-1.rds.amazonaws.com port=5432 dbname=postgres user=postgres password=postgres")
    cur = conn.cursor()
    cur.execute("insert into zdptest (time) values (current_timestamp)")
    print(dt_now)
    time.sleep(1)
    conn.commit()
    cur.close()
    conn.close()
  except:
    pass
