Getting JRA Race Information Using netkeiba-scraper on EC2

Published on 2020/02/10 üìñ 1 min read

Introduction

Using netkeiba-scraper on EC2 to retrieve race information. Since my goal here is data collection, I’ll go through the process up to storing the data in race.db.

Pre-setup

Installing Java and Git

sudo yum -y install java git

Since this is Amazon Linux, Corretto (a fully compatible open JDK) is installed.

Installing sbt

curl https://bintray.com/sbt/rpm/rpm | sudo tee /etc/yum.repos.d/bintray-sbt-rpm.repo
sudo yum -y install sbt

sbt Reference Manual ‚Äî Installing sbt on Linux https://www.scala-sbt.org/1.x/docs/ja/Installing-sbt-on-Linux.html

Installing Scala

Download from the official site .

wget https://downloads.lightbend.com/scala/2.13.1/scala-2.13.1.rpm
sudo rpm -ivh scala-2.13.1.rpm

Downloading and Configuring netkeiba-scraper

git clone https://github.com/stockedge/netkeiba-scraper.git

Configuration Changes

Modifying ./netkeiba-scraper/src/main/scala/netkeiba-scraper/Main.scala

  1. Change http to https

  2. Change the login URL: driver.get(“https://regist.netkeiba.com/account/?pid=login" )

  3. Change the collection period

    Collecting the default 10 years of data would take several days, possibly close to a week, so change it to 1 year.

    • Before modification
    // Collect URLs for the past 10 years
RaceListScraper.scrape(period = 12 * 10)
    • After modification
    // Collect URLs for the past 1 year
RaceListScraper.scrape(period = 12 * 1)

About the Data Stored in race.db

The created race.db has 4 tables:

  • feature
  • payoff
  • race_info
  • race_result

Note: there is also a sqlite_sequence table, but that’s for managing AUTOINCREMENT information, so the above 4 are the main ones.
