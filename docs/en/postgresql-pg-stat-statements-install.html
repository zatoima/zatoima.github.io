How to Install and Configure PostgreSQL's pg_stat_statements

Published on 2020/01/30 üìñ 1 min read
PostgreSQL

The contrib extension module includes pg_stat_statements. By installing and configuring this module, you can identify the following types of SQL. It can be used similarly to Oracle Database’s v$sql.

  • Executed SQL
  • SQL with long execution time
  • SQL with high execution count
  • SQL with low cache hit ratio

Type "help" for help.

Prerequisites

postgresql-contrib must be installed

-bash-4.2$ rpm -qa | grep contrib
postgresql10-contrib-10.11-2PGDG.rhel7.x86_64

Check the library

-bash-4.2$ pg_config --libdir
/usr/pgsql-10/lib
-bash-4.2$ find /usr/pgsql-10/lib -name pg_stat_statements.so
/usr/pgsql-10/lib/pg_stat_statements.so

How to Configure pg_stat_statements

As a prerequisite, the postgresql-contrib package (which includes the pg_stat_statements module) must be installed.

vi /var/lib/pgsql/10/data/postgresql.conf

Modify the following parameters:

shared_preload_libraries = 'pg_stat_statements'
pg_stat_statements.max = 1000
pg_stat_statements.track = top
pg_stat_statements.save = on

‚Äª Setting custom_variable_classes is not necessary. It was required up to version 9.2.

The description of each parameter is as follows. The main parameters to adjust are pg_stat_statements.max (to increase or decrease the number of SQL statements saved/stored) and pg_stat_statements.track (to determine whether recursive SQL should also be saved).

Parameter Description
shared_preload_libraries Specifies one or more shared libraries to be preloaded when the server starts. In this case, we need to load the pg_stat_statements library (/usr/pgsql-10/lib/pg_stat_statements.so), so pg_stat_statements is specified.
pg_stat_statements.max The default is 1000 SQL statements recorded. Specifies the maximum number of SQL statements to record. If more distinct SQL statements are detected, the information for the least-executed SQL statement will be discarded.
pg_stat_statements.track Controls which SQL statements are tracked by the module. top: Record top-level SQL statements (directly issued by clients) (default). all: Also record nested statements (e.g., called from within functions). none: Disable collecting statistics on statements.
pg_stat_statements.track_utility Specifies whether the module records utility commands. Utility commands are everything other than SELECT, INSERT, UPDATE, and DELETE. Default is on. This setting can only be changed by superusers.
pg_stat_statements.save Specifies whether to save statement statistics when the server shuts down. If off, statistics are not saved at shutdown and not reloaded at server startup. Default value is on.

A server restart is required after setting the parameters.

/usr/pgsql-10/bin/pg_ctl stop --pgdata=/var/lib/pgsql/10/data
/usr/pgsql-10/bin/pg_ctl start --pgdata=/var/lib/pgsql/10/data

Supplement

For RDS and Aurora, no library or parameter configuration is needed; just run CREATE EXTENSION pg_stat_statements;.

https://docs.aws.amazon.com/ja_jp/AmazonRDS/latest/UserGuide/CHAP_PostgreSQL.html

> Performance Metrics ‚Äì The pg_stat_statements module is included in shared_preload_libraries by default. So you don’t need to restart the instance immediately after creation. However, if you want to use this feature, you need to run the statement CREATE EXTENSION pg_stat_statements;. Also, track_io_timing is enabled by default so that detailed data can be added to pg_stat_statements.
