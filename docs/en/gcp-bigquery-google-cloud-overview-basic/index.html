Slots

Slots represent the degree of processing parallelism, with a default limit of 2000. BigQuery’s high-speed parallel processing is achieved through storage distribution and slot distribution, but note that the number of slots doesn’t necessarily scale up to this limit. It doesn’t seem to directly correspond to CPU core count.

https://cloud.google.com/bigquery/docs/slots?hl=ja

BigQuery slots are virtual CPUs used by BigQuery to execute SQL queries. BigQuery automatically calculates the number of slots needed for each query based on the size and complexity of the query.

The Unknown World of Google BigQuery - Qiita https://qiita.com/AkiQ/items/9c5eefb7953409aa2eda

As mentioned earlier, by default a maximum of 2,000 slots are given per project. The blazing speed of queries in BigQuery is achieved by the parallel processing of slots. Slots are allocated from BigQuery’s available resources at that point in time - which makes sense. Slots are basically global resources. This means that while it says you can use up to 2,000 slots, it doesn’t mean you can always use 2,000 slots simultaneously.
