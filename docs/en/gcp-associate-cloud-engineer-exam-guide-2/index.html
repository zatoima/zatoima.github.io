GCP Associate Cloud Engineer Exam Guide - Planning and Configuring Cloud Solutions

Published on 2021/09/07 üìñ 1 min read
GCP

Introduction

In preparation for the GCP Associate Cloud Engineer exam, I review the manuals listed in the exam guide and conduct some hands-on verification to study according to the exam guide. At the major section level, the following 5 topics are in the exam scope:

  1. Setting up a cloud solution environment
  2. Planning and configuring a cloud solution
  3. Deploying and implementing a cloud solution
  4. Ensuring successful operation of a cloud solution
  5. Configuring access and security

The Certification Exam Guide further subdivides these major sections, and each one seems to have specific exam topics broken down as follows.

This time, let’s review Planning and configuring a cloud solution.

2. Planning and configuring a cloud solution

2.1 Planning and estimating GCP product use using the Pricing Calculator

  • Use Google Cloud Pricing Calculator to make estimates

    Reference: Google Cloud Platform Pricing Calculator

2.2 Planning and configuring compute resources. Considerations include:

  • Selecting appropriate compute choices for a given workload (e.g., Compute Engine, Google Kubernetes Engine, App Engine, Cloud Run, Cloud Functions)

    • There are many hosting options. You need to select the appropriate compute service based on use cases such as deployment format, disk requirements, and scaling speed.

    Reference: Application Hosting Options | Hosting Options | Google Cloud

  • Using preemptible VMs and custom machine types as appropriate

    • Preemptible VM Instances

      Preemptible VMs are instances that you can create and run at a much lower price than normal instances. However, Compute Engine might stop (preempt) these instances if it needs to reclaim those resources for other tasks. Preemptible instances use Compute Engine’s excess capacity, and the availability of this capacity varies with usage.

      • Similar to AWS Spot Instances
        • Preemptible VM Instances | Compute Engine Documentation | Google Cloud

    • Custom Machine Types

      If predefined machine types don’t fit your needs, you can create a VM instance with custom virtual hardware settings. Specifically, you can use custom machine types to create VM instances with customized vCPU count and memory size.

      • Creating a VM Instance with a Custom Machine Type | Compute Engine Documentation | Google Cloud

      • When creating with gcloud commands, use --custom-cpu and --custom-memory options:

      gcloud compute instances create example-instance --custom-cpu=4 --custom-memory=5

      • It seems you can also add memory to existing instances, which I thought was a convenient feature.

        Each machine type has a specific amount of memory set by default depending on the machine. For example, if you create a custom N1 VM, you can set up to 6.5 GB of memory per vCPU. For a custom N2 VM, this number is raised to a maximum of 8 GB of memory per vCPU.

2.3 Planning and configuring data storage options. Considerations include:

  • Product choice (e.g., Cloud SQL, BigQuery, Cloud Spanner, Cloud Bigtable)

    • Consider the appropriate data store based on requirements such as whether transactions are needed, whether horizontal scaling is needed, whether SQL-based analytical queries are needed, whether a Key-Value type data store for large-scale low-latency workloads is needed, or whether data needs to span worldwide regions.
      • Google Cloud Databases

  • Choosing storage options (e.g., Standard, Nearline, Coldline, Archive)

    Storage Class API and gsutil name Minimum Storage Duration Typical Monthly Availability
    Standard Storage STANDARD None Multi-region and dual-region: >99.99%, Region: 99.99%
    Nearline Storage NEARLINE 30 days Multi-region and dual-region: 99.95%, Region: 99.9%
    Coldline Storage COLDLINE 90 days Multi-region and dual-region: 99.95%, Region: 99.9%
    Archive Storage ARCHIVE 365 days Multi-region and dual-region: 99.95%, Region: 99.9%

    Storage Classes | Cloud Storage | Google Cloud

2.4 Planning and configuring network resources. Tasks include:

  • Differentiating load balancing options

Reference: Choosing a Load Balancer | Load Balancing | Google Cloud

  • Identifying resource locations in a network for availability

    • Select locations taking into account the following factors. Make appropriate choices based on requirements such as deploying to multiple regions for load balancing, implementing a distributed front-end across multiple regions while keeping the backend in a single region, etc.
      • Region-specific limitations
      • User latency by region
      • Application latency requirements
      • How much control over latency
      • Balance between low latency and simplicity

    Reference: Best Practices for Compute Engine Region Selection | Solutions | Google Cloud

  • Configuring Cloud DNS

    • Cloud DNS is a high-performance, resilient global Domain Name System (DNS) service.
    • The GCP-specific terminology doesn’t immediately click, but it basically has similar characteristics and functionality to Route53.

Reference: Cloud DNS Overview | Google Cloud
