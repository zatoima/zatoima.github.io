Checking the Mapping Between Availability Zone ZoneNames and ZoneIds

Published on 2020/10/22 üìñ 1 min read
AWS

Availability Zones have the concepts of ZoneName and ZoneId, and as stated in the documentation, these mappings differ per account. For example, ap-northeast-1a might have a ZoneId of apne1-az4 for one account, and apne1-az1 for another. Without this specification, resources would tend to concentrate in ap-northeast-1a.

Resource AZ IDs - AWS Resource Access Manager https://docs.aws.amazon.com/ram/latest/userguide/working-with-az-ids.html

To ensure resources are spread across Availability Zones in a Region, Availability Zones are mapped to names individually for each account. For example, the Availability Zone us-east-1a in your AWS account might not be in the same location as us-east-1a in another AWS account.

Regions and Zones - Amazon Elastic Compute Cloud https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/using-regions-availability-zones.html

To coordinate Availability Zones across accounts, you must use the AZ ID, which is a unique and consistent identifier for an Availability Zone. For example, use1-az1 is an AZ ID for the us-east-1 Region and maps to the same location in every AWS account.

Viewing AZ IDs enables you to determine the location of resources in one account relative to the resources in another account. For example, if you share a subnet in an Availability Zone with AZ ID use-az2 with another account, this subnet is available to that account in the Availability Zone whose AZ ID is also use-az2. The AZ ID for each VPC and subnet is displayed in the Amazon VPC console.

You can check using this command (aws ec2 describe-availability-zones). The ZoneIDs for my account were as follows:

ZoneName ZoneId
ap-northeast-1a apne1-az4
ap-northeast-1c apne1-az1
ap-northeast-1d apne1-az2
