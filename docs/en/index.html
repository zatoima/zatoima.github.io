### Step 1: Identify API Recording Trends in CloudTrail Using Athena

Identify the event sources and check if any specific service has an unusually high number of events.

[Analyzing CloudTrail Trails with Athena \| my opinion is my own](https://zatoima.github.io/aws-cloudtrail-athena-analyze-query/)

```sql
SELECT
    awsregion,
    eventsource,
    eventname,
    COUNT(*) as cnt
FROM
    cloudtrail_logs
WHERE
    date>='2021/11/24' AND
    region = 'ap-northeast-1'    
GROUP BY
    awsregion,
    eventsource,
    eventname
ORDER BY cnt DESC
limit 10;

Next, drill down into the prominent event sources and event names. The UserIdentity element contains requester information, which helps identify the cause.

Step 2: Reconsider and Narrow Down Data Event Delivery and CloudTrail Insights Delivery

The first copy of management events is free, but data event and Insights delivery are charged, so be careful. Data events in particular tend to generate a high number of API calls.

Feature Pricing
Management events delivered to S3 The first copy of management events is delivered free of charge. Additional copies: 2.00 USD per 100,000 management events delivered
Data events delivered to S3 0.10 USD per 100,000 data events delivered
CloudTrail Insights 0.35 USD per 100,000 events analyzed
  • Pricing - AWS CloudTrail | AWS

You can also use the event selector feature to filter which data events to record, so design your configuration to exclude unnecessary data events.
