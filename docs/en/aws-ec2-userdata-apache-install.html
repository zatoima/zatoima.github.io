Auto-Installing Apache on EC2 Using User Data

Published on 2020/08/31 üìñ 1 min read
AWS EC2

What is User Data?

User data refers to commands that run on a Linux instance at initial startup. By default, they only run at the initial startup, but it is also possible to run them on every reboot.

https://docs.aws.amazon.com/en_us/AWSEC2/latest/UserGuide/user-data.html

When you launch an instance in Amazon EC2, you have the option of passing user data to the instance that can be used to perform common automated configuration tasks and even run scripts after the instance starts. You can pass two types of user data to Amazon EC2.

How to Configure User Data

There is a field to specify user data under Advanced Details in Step 3: Configure Instance Details.

Specify the following in user data. This performs yum update, Apache installation, permission changes, and more.

#!/bin/bash
yum update -y
amazon-linux-extras install -y lamp-mariadb10.2-php7.2 php7.2
yum install -y httpd mariadb-server
systemctl start httpd
systemctl enable httpd
usermod -a -G apache ec2-user
chown -R ec2-user:apache /var/www
chmod 2775 /var/www
find /var/www -type d -exec chmod 2775 {} \;
find /var/www -type f -exec chmod 0664 {} \;
echo `hostname` > /var/www/html/index.html

After that, proceed with creating the instance as normal, and Apache will be installed automatically when the instance is created. Security group settings and other configurations are omitted here.
