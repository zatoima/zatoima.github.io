Running HammerDB TPC-C Against Aurora PostgreSQL

Published on 2020/11/13 üìñ 2 min read
AWS Aurora PostgreSQL

I tried HammerDB’s TPC-C against Aurora, and as expected, the procedure is almost the same as regular PostgreSQL.

Introduction

Using the CLI, referencing the following HammerDB documentation:

Chapter 8. Command Line Interface (CLI) https://hammerdb.com/docs/ch08.html

Prerequisites

Assumes PostgreSQL-related libraries are already installed. If not, install them with:

sudo yum -y install postgresql
sudo yum -y install postgresql-contrib

Download the HammerDB Installer

wget https://github.com/TPC-Council/HammerDB/releases/download/v3.3/HammerDB-3.3-Linux.tar.gz
tar xvzf HammerDB-3.3-Linux.tar.gz
cd HammerDB-3.3

HammerDB https://hammerdb.com/download.html

HammerDB Parameter Configuration

Set the Target DB to PostgreSQL
hammerdb> dbset db pg
Set the Benchmark to TPC-C
hammerdb> dbset bm TPC-C
Set the Target Hostname
hammerdb> diset connection pg_host aurorapgsqlv1.cluster-xxxxxxxxx.ap-northeast-1.rds.amazonaws.com
Set the Superuser Username and Password
hammerdb> diset tpcc pg_superuser postgres
hammerdb> diset tpcc pg_superuserpass postgres
Change pg_driver

Specify test for setup and verification, and timed for actual measurement

hammerdb> diset tpcc pg_driver timed
Change pg_timeprofile

When set to true, response time percentiles at 10-second intervals and cumulative values at completion are reported

hammerdb> diset tpcc pg_timeprofile true
Change pg_total_iterations

Specify the number of transaction executions (default: 1000000)

hammerdb> diset tpcc pg_total_iterations 100000
Set pg_count_ware (Scale Factor)

Adjust the data volume.

hammerdb> diset tpcc pg_count_ware 10
Set pg_num_vu (Concurrent Users)

Adjust the data volume.

hammerdb> diset tpcc pg_num_vu 4
Verify Configuration Values
