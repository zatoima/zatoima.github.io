Installing GoldenGate and Building a Replication Environment Between Databases

Published on 2019/03/29 üìñ 4 min read
Oracle GoldenGate

We will set up replication between Oracle Databases using GoldenGate.

The steps to establish replication are as follows, but this article covers steps “1.” and “2.”:

  1. GoldenGate Installation

  2. GoldenGate Environment Configuration

  3. Data Propagation

Environment Configuration

Due to machine resource constraints, two databases are created on the same machine. We will use GoldenGate to create a replication environment between these two databases.

Source Target
DB Version 11.2.0.4 18c
GoldenGate Version 12.2.0.1 18.1.0.0.0
DB Name db112s db18s
PDB Name - db18p1
GoldenGate DB User ggs c##ggs
Propagation User ggtest ggtest
GoldenGate Install Dir /gg/gg1 /gg/gg2
Capture Process c11 -
Data Pump Process d11 -
Replicat Process - r11
Capture Process Mode Integrated Mode(Integrated Capture) -
Replicat Process Mode - Integrated Mode(Integrated Replicat)
Other Single Instance

Prerequisites

  1. Source and target databases are already installed
  2. GoldenGate media files are downloaded in advance

Create Trail File Storage Directories

“/gg/gg1/c11” is the storage directory for the Local Trail generated by the source Capture process, and “/gg/gg2/d11” is the storage directory for the Remote Trail generated by the Data Pump on the target side.

mkdir -p /gg/gg1/dirdat/c11 #Storage directory for Local Trail
mkdir -p /gg/gg2/dirdat/d11 #Storage directory for Remote Trail

Database Environment Changes (Source Side)

Create Tablespace

. /home/oracle/.oraenv_db112s --load environment variables
sqlplus / as sysdba
create tablespace ggdata datafile '/u01/app/oracle/oradata/DB112S/datafile/ggdata.dbf' size 5G;

Create Propagation Target User and GoldenGate Admin User

CREATE USER ggs IDENTIFIED BY oracle DEFAULT TABLESPACE ggdata;
CREATE USER ggtest IDENTIFIED BY oracle DEFAULT TABLESPACE ggdata;

GRANT dba TO ggs;
GRANT dba TO ggtest;
exec dbms_goldengate_auth.grant_admin_privilege('GGS');

Change Initialization Parameters / Set Minimum Supplemental Logging

-- Pre-check
show parameter enable_goldengate_replication
select name,supplemental_log_data_min from v$database;
select log_mode from v$database;

-- Change initialization parameters
alter system set enable_goldengate_replication=true scope=both;
alter system set streams_pool_size=1250M scope=both SID='*'; -- Integrated Capture requirement

-- Set minimum supplemental logging
alter database add supplemental log data;
alter system switch logfile;

-- Verify settings
show parameter enable_goldengate_replication
select name,supplemental_log_data_min from v$database;
select log_mode from v$database;
exit

Create GoldenGate Processes

Create Source Side Processes

. /home/oracle/.oraenv_db112s --load environment variables
cd /gg/gg1
./ggsci

#DB login
dblogin userid ggs@db112s,password oracle

#Create and add Capture process
REGISTER EXTRACT c11 DATABASE
add extract c11, integrated,tranlog, begin now
add exttrail  ./dirdat/c11/lt, extract c11, megabytes 500

#Create and add Data Pump process
add extract d11, exttrailsource ./dirdat/c11/lt
add rmttrail ./dirdat/d11/rt, extract d11

#Check status
info all

Create Target Side Processes

Since Replicat applies to the PDB, the connection descriptor specified when logging in with dblogin must point to the target PDB, not CDB$ROOT.

Also, while Integrated Capture requires the register command, Integrated Replicat does not (it is issued internally).

. /home/oracle/.oraenv_db18s --load environment variables
cd /gg/gg2
./ggsci

#DB login
dblogin userid c##ggs@db18p1,password oracle

#Create Replicat process
ADD REPLICAT r11,integrated EXTTRAIL ./dirdat/d11/rt

#Check status
info all

Appendix

OGG-02912 Error Resolution

With the combination of DB11.2 and GG12.2 (Integrated Capture), OGG-02912 may occur and the Capture process may ABEND on first startup (depends on the PSU/BP applied).

This error occurs because the Trail file format was changed starting from GoldenGate 12.2. (It occurs when the database-side LogMiner function does not support the new format.)

ERROR OGG-02912 Patch 17030189 is required on your Oracle mining database for trail format RELEASE 12.2 or later.

If this error occurs, you can either apply “Patch 17030189” as stated in the error message, or run “prvtlmpg.plb” located under GG_HOME to work around the error.

. /home/oracle/.oraenv_db112s --load environment variables
cd /gg/gg1
sqlplus / as sysdba
@prvtlmpg.plb
