Setting Up a Spark on Docker on EC2 Environment

Published on 2022/02/04 📖 1 min read
Docker Spark AWS

When I use a Spark environment, I typically rely on AWS EMR. However, it requires at least three xlarge EC2 instances and takes roughly 10 minutes to provision. So I created a Spark on Docker on EC2 environment for quickly testing PySpark scripts.

Navigate to the Jupyter Console

If a Public IP or Elastic IP is assigned to the instance, make sure to allow access through the security group settings.

http://<Public IP>:8888/lab?token=65c095c6ac9d7bf2d3eec7189bc1c0e8e5c254f1aad33689

Click on “Terminal” under the “Other” section to start using the PySpark environment.
