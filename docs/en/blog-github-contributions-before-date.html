Creating GitHub Contributions with Past Dates

Published on 2020/01/13 üìñ 1 min read
Blog Github

Introduction

This blog is hosted on GitHub Pages, and one of my goals for this year is to “regularly update the blog and produce output,” so I wanted to visualize that. GitHub has a “grass” (contribution graph) that visualizes contributions, so I decided to use that for visualization.

On the other hand, I had many articles updated before I started using GitHub Pages, so I wanted to reflect those update dates in the contributions. I looked into how to create GitHub contributions with past dates.

Method

The conditions for contributions to show up are as follows. Commits, issues, and pull requests count as contributions. No source code changes are required‚Äîthis alone is enough to fake create contributions.

https://help.github.com/en/github/setting-up-and-managing-your-github-profile/viewing-contributions-on-your-profile

What counts as a contribution

On your profile page, certain actions count as contributions:

  • Committing to a repository’s default branch or gh-pages branch
  • Opening an issue
  • Proposing a pull request
  • Submitting a pull request review

This time, I specified --date in the git commit command so that the AuthorDate would be a past date. Then just push and you’re done.

Command
git commit --allow-empty -m "COMMIT" --date="Dec 29 23:59:59 2019 +0900"
git push origin master
Verification
git log --pretty=fuller

commit d0d02bd117419f4aacc7d8f16e78f192ec8ce79d
Author:     Imazato <xxxxx.xxxxxx@gmail.com>
AuthorDate: Sun Dec 29 23:59:59 2019 +0900
Commit:     Imazato <xxxxx.xxxxxx@gmail.com>
CommitDate: Wed Jan 8 17:46:25 2020 +0900

    COMMIT
Result

By pulling the update dates from the articles and running the command, contributions were successfully created even for past dates. I aim to become someone who can’t relax without seeing green on the contribution graph.
