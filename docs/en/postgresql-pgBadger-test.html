Trying pgBadger, a PostgreSQL Log Analysis Tool

Published on 2020/01/07 üìñ 1 min read
PostgreSQL

Introduction

pgBadger is a tool that analyzes PostgreSQL log files and outputs reports. It analyzes logs to provide information on SQL, wait events, I/O statistics, error statistics, SQL histograms, locks, vacuum statistics, and many other workloads.

If you want to see a sample report first, check here . It was more graphical than I expected.

The GitHub repository and README are here.

darold/pgbadger: A fast PostgreSQL Log Analyzer https://github.com/darold/pgbadger

Since pgBadger can statically analyze logs if installed on EC2, it can apparently be run against RDS or Aurora PostgreSQL-compatible instances, so let’s check it out. (Note: pgbadger was run on PostgreSQL 10.11 installed on EC2.)

Optimizing and Tuning Queries in Amazon RDS PostgreSQL Based on Native and External Tools | Amazon Web Services Blog https://aws.amazon.com/jp/blogs/news/optimizing-and-tuning-queries-in-amazon-rds-postgresql-based-on-native-and-external-tools/

pgBadger Configuration and postgres.conf Settings

Installation
[ec2-user@postdb ~]$ sudo yum install pgbadger
[ec2-user@postdb ~]$ which pgbadger
/usr/bin/pgbadger
[ec2-user@postdb ~]$
postgres.conf

Modify the log-related parameters. Note that this will output more information than the defaults.

log_filename = 'postgresql-%Y-%m-%d.log'
# Log query text and execution time for queries exceeding the specified time. 0 means all queries are targeted; adjust if you want to filter.
log_min_duration_statement = 0
log_line_prefix = '%t [%p]: [%l-1] user=%u, db=%d'
# Log checkpoint execution
log_checkpoints = on
# Log client connections
log_connections = on
# Log client disconnections
log_disconnections = on
# Log lock waits exceeding deadlock_timeout (default 1 second)
log_lock_waits = on
# Log when temporary files exceed the specified size
log_temp_files = 0
lc_messages = 'C'
log_autovacuum_min_duration = 0
log_error_verbosity = default

Execution Command

/usr/bin/pgbadger -f '%t [%p]: [%l-1] user=%u, db=%d' -I -q /var/lib/pgsql/10/data/log/postgresql-2019-12-30.log -O /var/lib/pgsql/pgbadger/

Information That Can Be Obtained

You can view statistics by time period, SQL queries with the most waits, queries with the highest execution count, and rankings of queries with the longest average execution time. For a sample report, see here .

https://github.com/darold/pgbadger#FEATURE

Other Features

  • Automatic incremental report mode (run via cron with the incremental option)
  • Monthly reports
  • Parallel processing
