Performing Failover of Aurora PostgreSQL Using RDS Proxy

Published on 2020/08/29 üìñ 2 min read
AWS Aurora PostgreSQL

The goal here is to explore RDS Proxy and observe how long failover takes and how it behaves.

With the traditional cluster endpoint, Writer and Reader switching is implemented using DNS. With RDS Proxy, the connection between the application and RDS Proxy is maintained while the DB-side failover is detected and connections are switched seamlessly. Applications no longer need to handle reconnection or DNS TTL considerations. Connections in the middle of a transaction or query processing will still require reconnection.

Introduction to Amazon RDS Proxy

https://pages.awscdn.com/rs/112-TZM-766/images/EV_amazon-rds-aws-lambda-update_Jul28-2020_RDS_Proxy.pdf

So let’s perform failover with both patterns: using the cluster endpoint and using RDS Proxy.

Pattern 1: Using Cluster Endpoint

Run the following script to record timestamps in a table, and manually trigger a failover while the script is running.

Test Script

INSERT every 1 second

#! /bin/bash

while true
do
  #truncate table record;
  #create table record(id serial,create_at timestamp);
  DATETIME=$(date -u "+%Y-%m-%d %T.%N")
  echo $DATETIME
  psql -q -h a-pgsql.cluster-xxxxx.ap-northeast-1.rds.amazonaws.com -d postgres -U postgres -c "insert into record(create_at) VALUES (current_timestamp)" &
  sleep 1
done

Pattern 2: Using RDS Proxy

Run the same procedure as Pattern 1 to check the failover time and DB connection status. Specify the RDS Proxy endpoint as the connection target.

Script

#! /bin/bash

while true
do
  #truncate table record;
  #create table record(id serial,create_at timestamp);
  DATETIME=$(date -u "+%Y-%m-%d %T.%N")
  echo $DATETIME
  psql -q -h proxy-aurora.proxy-xxxxx.ap-northeast-1.rds.amazonaws.com -d postgres -U postgres -c "insert into record(create_at) VALUES (current_timestamp)" &
  sleep 0.5
done
